GIT

  • git config --global user.name blabla - вказати ім'я
  • git config --global user.email blabal@bla - вказати email
  • git init - ініцілізувати/створити новий репозиторій
  • git status - показує інформацію про файли, зміни та створення комітів
  • git add - добавляє файл в рипозиторій
  • git commit - бере всі дані, добавлені в індекс за допомогою git add,зберігає їх зліпок,записує зміни в репозиторій.
  • git log - історія коммітів в данній branch/вітці
  • git checkout - переключення між двома вітками, відновлення старої версії.
  • git branch - вітки гіта,створити нову вітку
  • git push - відправляє всі зміни на віддалений репозиторій
  • git pull - отримати дані з віддаленого репозиторію та обєднати з локальною віткою
  • git merge - вносить зміни з віддаленого репо на локальний, зливає одну або декілька віток в поточну
  • git clone - завантажує репозиторій разом з усією його історією
  • .gitignore - використвовується для вказівки в ньому файлів чи папок,які потрібно приховати від системи контролю версій, допомагає застосовувати один комміт з одніжї вітки до дерева іншої
  • git revert - створює новий комміт на скасовує попередні
  • git reset - відміна всіх коммітів після вказаного комміта,але залишає всі зміни в робочій дерикторії
  • git rebase - один з двох способів об'єднати змінни, зроблені в одній гілці,з іншою гілкою
  • git cherry-pick - використовується для того, щоб взяти зміни,внесені кимось та спробувати застосувати їх заново у виді нового комміта
  • git stash - збереження та відновлення незавершенних змін
